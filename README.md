# 🧪 Econic Telegram Mini App - QA Portfolio

Welcome to my QA Portfolio! This repository contains quality assurance documentation, test cases, and bug reports for **Econic**, an eco-themed Telegram Mini App. 

As a QA Contributor for this project, my goal is to ensure a seamless, engaging, and bug-free user experience by conducting manual testing, identifying software defects, and closely collaborating with the product development team.

## 📁 Repository Contents

This repository is structured to demonstrate the end-to-end manual testing process:

*   **[Test Cases](./Test_Cases.md):** Detailed functional and UI/UX test scenarios for core game mechanics and user onboarding.
*   **[Bug Reports](./Bug_Reports.md):** Examples of documented software defects, including steps to reproduce, expected/actual results, and severity levels.
*   **[Regression Checklist](./Regression_Checklist.md):** A streamlined checklist used to verify that new updates do not break existing functionality.
*   **[Test Summary Report](./Test_Summary.md):** A brief overview of testing cycles, highlighting overall product quality and usability improvements.

## 🛠️ Tools & Skills Demonstrated

*   **Testing Types:** Manual Testing, Functional Testing, Regression Testing, User Acceptance Testing (UAT).
*   **Platform:** Telegram Mini Apps (Mobile and Desktop environments).
*   **Documentation:** Test Case Design, Defect Tracking, Requirements Analysis.
